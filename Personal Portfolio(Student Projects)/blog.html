How I Plan Projects

Every project begins with a simple plan: sketch the interface, list required features, and break tasks into small commits. For class mini-projects, I create a short README so instructors can run the project locally and review changes on GitHub. I keep notes on what went well and what I learned. This blog page contains quick posts about debugging tips, design decisions, and commands I used in Git. These short reflections help me prepare for live grading sessions and show continuous progress. I often include screenshots and code snippets when relevant so peers and instructors can follow my work more easily.

Post: Adding a Contact Form (frontend only)

I added a simple contact form to my portfolio. Since there's no backend for this class activity, the form shows a friendly confirmation message after submission. This satisfies the interactivity requirement and demonstrates basic form handling with JavaScript. For future improvements, I plan to learn how to connect a lightweight backend or use email services if allowed.
